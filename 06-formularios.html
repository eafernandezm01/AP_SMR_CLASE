FORMULARIOS

FORMULARIO

Para definir un formulario en HTML usamos la etiqueta <form>

Algunos de sus atributos más importantes:

  • action: define la acción que se realizará cuando se envíe el formulario.
  • target: dónde muestra la respuesta. (_blank, _self...). Por defecto, _self
  • method: Los datos del formulario se pueden enviar como:
    • method="get": Por defecto, los parámetros van por la URL: https://www.miweb.com/alta.php?dni=12345678Z&nombre=UsuarioX
    • method="post": Internamente, no se muestran en la url.

Dentro del formulario podemos inluir:

  • <label>: Define una etiqueta, normalmente, va asociada a otro elemento, esta asociación se realiza con el atributo id (atributo único).
  • <input>: Caja para introducir datos en una línea.
  • <textarea>: Caja para introducir datos en varias líneas.
  • <button>: Botón.
  • <select>: Lista desplegable
  • <fieldset>: Agrupa los elementos anteriores y se pone un título con <legend>.

INPUT

Existen numerosos tipos de <input>, si pones en Visual Studio Code "in", te saldrá muchas opciones...

  • <input type="text"> define un campo de entrada de texto de una sola línea.
  • <input type="password"> define un campo de contraseña.
  • <input type="search"> define un campo de búsqueda (es como un campo normal sólo que tiene una x al final para borrar los datos)
  • <input type="hidden"> define un campo de entrada oculto. No se ve a simple vista (en la página pero sí se ve en las herramientas de desarrollo en el fuente".
  • <input type="date"> define un campo de entrada fecha.
  • <input type="time">define un campo de entrada hora.
  • <input type="datetime-local"> define un campo de entrada fecha - hora.
  • <input type="month"> define un campo de entrada mes - año.
  • <input type="week"> define un campo de entrada semana - año.
  • <input type="email"> define un campo de entrada email. Valida que tenga @ al enviar el formulario (submit).
  • <input type="number"> define un campo de entrada numérico. No permite introducir letras. Se puede establecer un número mínimo y máximo (atributos min y max).
  • <input type="tel"> define un campo de entrada teléfono. Se puede validar teléfono estableciendo una expresión regular (atributo pattern).
  • <input type="range"> define un campo de entrada númerico mediante un control deslizante. Se puede establecer un número mínimo y máximo (atributos min y max) y step para definir el valor de incremento.
  • <input type="color"> define una paleta de colores.
  • <input type="url"> define un campo de entrada url. Algunos navegadores son restrictivos a la hora de validar la url.
  • <input type="file"> define un campo de entrada para añadir un fichero.
  • <input type="radio"> define un botón de radio.
  • <input type="checkbox"> define una casilla de verificación (puedes pinchar en 0 o más de una opción)
  • <input type="submit"> define un botón para enviar (submit) datos de formulario al servidor (donde se especifique en la etiqueta action del form)
  • <input type="image"> define una imagen como botón para enviar (submit). En el atributo src se define la ruta de la imagen (local o remota).
  • <input type="button"> define un botón. No hace submit del formulario.
  • <input type="reset"> restablecerá todos los valores del formulario a sus valores predeterminados.

¡Por cierto es una etiqueta que no tiene cierre!

TEXTAREA

Campo que permite múltiples líneas, es usado para las descripciones o textos largos. Se puede especificar el tamaño por CSS o atributos (col="20" row="20")

<textarea name="desc" rows="4" cols="40"></textarea>

DATALIST

Permiten buscar al introducir un valor en el input. Se suelen usar cuando tenemos muchos valores posibles. Se define un input con el atributo list="idDelDataList" que debe tener el mismo nombre que el id del datalist.

<input list="browsers" name="browser" id="browser">
<datalist id="browsers">
 <option value="Edge">
 <option value="Firefox">
 <option value="Chrome">
 <option value="Opera">
 <option value="Safari">
</datalist>

SELECT

Usamos la etiqueta: <select> que permite definir una lista desplegable con las opciones marcadas como option.
Por defecto al cargar la página, pone como seleccionada la primera, en el caso en que se quiera cambiar, debemos añadir el atributo selected en el option que queramos.
Los select pueden ser simples (sólo se puede elegir un option) o múltiples (1 o varios).

Sólo podemos seleccionar un valor:

<select id="cars">
 <option value="volvo">Volvo</option>
 <option value="saab">Saab</option>
 <option value="opel">Opel</option>
 <option value="audi">Audi</option>
</select>

Para que permita elegir varios elementos debemos añadir en la etiqueta select multiple

<select id="cars" multiple>
 <option value="volvo">Volvo</option>
 <option value="saab">Saab</option>
 <option value="opel">Opel</option>
 <option value="audi">Audi</option>
</select>
